§5 — Dataset & Results

Dataset Description
& Benchmarks

Two-class dataset combining the NEU Surface Defect Dataset (Scratch class) with self-collected No-Defect images.

5.1 Dataset Overview

The dataset used in this project consists of two classes: Scratch (Defect) and No-Defect (Normal). This binary construction directly reflects the practical MSME inspection scenario — a production line system needs to answer one question: is this part defective or not?

Class 1 — Scratch (Defect)
NEU
Images sourced from the NEU Surface Defect Dataset — a publicly available benchmark containing labeled steel surface defect images. Only the Scratches (Sc) class is used. These images represent real industrial steel surfaces with visible scratch defects captured under controlled imaging conditions.
Class 0 — No-Defect (Normal)
Self
Since the NEU dataset contains only defective samples, defect-free metal surface images are self-collected to form the normal class. Captured using the same camera setup, viewing angle, and lighting conditions as the defect samples to ensure domain consistency.
Why self-collect the normal class? The NEU dataset provides no defect-free reference images — a critical gap for binary classification. Mixing unrelated public datasets would introduce domain mismatch. Self-collected images from the same setup ensure the classifier learns the correct surface texture of this workshop's components, not a different factory's steel.
5.2 Dataset Characteristics
Image Type
Grayscale / RGB. Converted to grayscale during preprocessing. Original NEU images are grayscale; self-collected images captured as RGB and converted.
Resolution
Uniformly resized to a fixed resolution (200×200 px) during preprocessing. Original NEU resolution is 200×200 px; self-collected images resized to match.
Classes
Class 0 — No-Defect (normal steel surface, no visible scratch, dent, or corrosion)  ·  Class 1 — Scratch (visible linear scratch defect on steel surface)
Conditions
Realistic workshop lighting and surface variations. No controlled laboratory conditions. Both classes captured under matching camera geometry, viewpoint, and approximate illumination.
Domain Consistency
Self-collected No-Defect images use the same camera setup, viewing angle, and lighting conditions as the NEU Scratch images — avoiding domain mismatch that would inflate or deflate classifier performance.
5.3 No-Defect Image Collection Protocol

The self-collected normal class follows a strict collection protocol to ensure domain consistency with the NEU scratch images. Surfaces selected are clean steel plates without visible scratches, dents, or corrosion.

Surface Selection
Clean steel plates without visible scratches, dents, rust spots, or surface contamination. Multiple plates used to capture natural variation in surface texture and finish.
Camera Setup
Same fixed overhead camera, same approximate distance, same viewing angle as used for any defect image captures. Consistent perspective across both classes.
Lighting Conditions
Realistic workshop lighting — not studio-controlled. Mirrors real MSME inspection conditions, including minor illumination variation between captures.
Validation
Each collected image manually verified to confirm absence of any visible defect before inclusion in the No-Defect class. Ambiguous surfaces discarded.
