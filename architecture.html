System Architecture

Modular Pipeline
Architecture

Domain-independent core with domain-specific model selection — the same preprocessing and feature extraction serves metal surfaces.

01 Design Principles
Key insight: Preprocessing and feature extraction are kept domain-independent. Only the final ML classifier is swapped per domain. This makes the framework extensible to new surfaces without redesigning the full pipeline.
Modularity
Each stage is a self-contained module with a defined input/output contract. Stages can be swapped or tuned independently without breaking the pipeline.
Interpretability-First
Every decision can be traced back to specific feature values. No black-box layers — suitable for certified quality control environments.
CPU-Native
No GPU dependencies. All operations — from Gabor filtering to Random Forest inference — run on standard multi-core CPUs available in factory settings.
Low-Data Regime
Architecture is designed for 100–500 samples per class, not thousands. PCA dimensionality reduction actively prevents overfitting on small datasets.
02 Full Pipeline Architecture
S1 ↓
Image Acquisition
Capture high-resolution surface images (≥2048×2048 px) under controlled 45° incident LED lighting to minimize specular reflection. Images sourced from the NEU Metal Surface Defect Dataset — 6 defect classes × 300 images each = 1800 total samples.
NEU Dataset 2048×2048 px 45° LED lighting Grayscale
S2 ↓
Preprocessing
Sequential noise reduction and contrast normalization pipeline. Gaussian blur removes high-frequency sensor noise. CLAHE (Contrast Limited Adaptive Histogram Equalization) enhances local contrast in defect regions. Bilateral filtering preserves edge sharpness while smoothing homogeneous surface regions. Final intensity normalization to [0,1].
Gaussian Blur CLAHE Bilateral Filter Normalization [0,1]
S3 ↓
Defect Segmentation
Otsu's global thresholding adaptively determines an optimal binarization threshold from the image histogram. Morphological opening removes small noise blobs; closing fills gaps within defect regions. Connected component analysis labels distinct candidate regions. Size filtering (minimum 50 px²) eliminates sensor artifacts.
Otsu Thresholding Morphological Open/Close Connected Components Size Filter ≥50px²
S4 ↓
Feature Extraction
Multi-modal feature extraction capturing complementary surface properties. LBP captures rotation-invariant micro-texture. Gabor filter bank extracts directional texture at multiple scales and orientations. GLCM computes second-order texture statistics. Shape descriptors characterize defect geometry. Statistical moments (mean, variance, skewness, kurtosis) describe intensity distributions.
LBP Texture Gabor 4×3 Bank HOG Gradients GLCM Shape Descriptors Statistical Moments
S5 ↓
Dimensionality Reduction
PCA reduces the raw 100+ dimensional feature vector to the 20–30 most discriminative principal components, retaining ≥95% of total variance. This eliminates correlated redundancy between features, accelerates downstream classification, and significantly reduces overfitting risk on small datasets. LDA further projects features to maximize between-class separation.
PCA → 20–30 components ≥95% variance retained LDA projection
S6
ML Classification
Domain-specific classifier is selected based on target material. For metal surfaces: LDA + Nearest Centroid classifier provides fast, interpretable decisions; Random Forest provides non-linear boundaries and feature importance rankings; SVM with RBF kernel serves as a strong baseline. Final output includes defect class label and confidence score.
LDA + Nearest Centroid Random Forest SVM-RBF Confidence Score
03 Data Flow Specifications
Input Format
Single-channel grayscale image, 200×200 px (resized from original). Pixel values: uint8 [0, 255].
After Preprocess
Float32 image, same spatial dimensions. Pixel values normalized to [0.0, 1.0].
After Feature Ext.
1D feature vector of ~120 dimensions: LBP histogram (59D) + Gabor responses (12D) + HOG descriptor (N) + GLCM stats (5D) + shape features (7D) + statistical moments (4D).
After PCA
Reduced vector of 20–30 float components, zero-mean, unit-variance in PCA space.
Output
Integer class label ∈ {0…5} mapping to defect category + float confidence ∈ [0,1] for the predicted class.
04 Classifier Comparison
LDA + Nearest Centroid
Primary production model. Computes class centroids in LDA-transformed space and assigns test samples by Euclidean distance. Extremely fast (<5ms), fully interpretable, no hyperparameters to tune.
Random Forest
Ensemble of 200 decision trees providing non-linear decision boundaries. Built-in feature importance ranking reveals which descriptors (LBP vs. Gabor vs. shape) contribute most to each defect class.
SVM (RBF Kernel)
Strong statistical baseline for high-dimensional feature spaces. RBF kernel handles non-linear class boundaries. Tuned via grid search over C and γ with 5-fold cross-validation.
k-NN (k=5)
Simplest baseline. Classifies by majority vote among 5 nearest neighbors in PCA space. Useful for anomaly detection and understanding the feature space structure.
